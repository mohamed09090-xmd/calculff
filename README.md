# Game Credit Profit Manager

تطبيق Flutter عربي RTL يعمل محليًا لإدارة حساب رصيد الألعاب، باقات الشراء، مخزون الرصيد وصلاحيته، وسجل الأرباح النقدية.

> التطبيق **لا يتصل بدجيزي أو أي لعبة**، ولا ينفذ شراءً أو دفعًا، ولا يستخدم scraping أو reverse engineering. كل العمليات يدوية ومحلية على الجهاز.

## الحالة الحالية

هذه نسخة MVP كاملة موجهة إلى Android، وتضم:

- حاسبة حسب مبلغ العميل، عدد الجواهر، أو الرصيد المطلوب.
- محسن باقات ديناميكي `PackageOptimizer` يدعم تكرار الباقة.
- ترتيب النتائج حسب: أقل تكلفة، أقل فائض، أقل عدد باقات، ثم أفضل صلاحية.
- منتجات وباقات قابلة للإضافة والتعديل والتعطيل.
- مخزون على شكل رزم مستقلة مع تاريخ شراء وانتهاء.
- استهلاك FEFO: الأقرب انتهاءً أولًا.
- حفظ Snapshot لبيانات المنتج والباقات داخل العملية التاريخية.
- حذف العملية مع إعادة بناء دفتر المخزون زمنيًا.
- Dashboard للمبيعات والأرباح والرصيد.
- سجل عمليات وتفاصيل كاملة.
- عرض المبالغ بالدينار أو بصيغة “الألف”.
- وضع فاتح وداكن.
- تصدير واستيراد نسخة JSON مع فحص الإصدار والبنية.
- تنبيهات محلية قبل انتهاء الرصيد.
- اختبارات Unit للحساب والتحسين وFEFO والتحويل النقدي.
- GitHub Actions للتحليل والاختبار وبناء APK من الهاتف فقط.

## التجربة باستخدام الهاتف فقط

الطريقة الموثوقة هي GitHub Actions، لأن بيئة بناء Flutter/Android الكاملة ليست مسارًا رسميًا بسيطًا داخل Android نفسه.

### 1. فك المشروع في Termux

```bash
pkg update
pkg install git unzip
termux-setup-storage
cp /sdcard/Download/game-credit-profit-manager-mvp.zip ~/
cd ~
unzip game-credit-profit-manager-mvp.zip
cd game_credit_profit_manager
```

### 2. ارفع المشروع إلى GitHub

أنشئ مستودعًا فارغًا من تطبيق GitHub أو الموقع، ثم داخل Termux:

```bash
git init
git add .
git commit -m "Initial Flutter MVP"
git branch -M main
git remote add origin https://github.com/USERNAME/REPOSITORY.git
git push -u origin main
```

استبدل `USERNAME` و`REPOSITORY`. عند طلب كلمة المرور استخدم Personal Access Token بدل كلمة مرور الحساب.

### 3. ابنِ APK من الهاتف

- افتح المستودع في GitHub.
- ادخل إلى **Actions**.
- افتح workflow باسم **Verify and build Android APK**.
- اختر **Run workflow** إن لم يبدأ تلقائيًا.
- بعد نجاحه افتح قسم **Artifacts**.
- نزّل `game-credit-profit-manager-debug-apk`.
- فك الملف وثبّت `app-debug.apk`.

ملف workflow موجود في:

```text
.github/workflows/build-apk.yml
```

وهو يشغّل بالترتيب:

```bash
flutter pub get
flutter analyze
flutter test
flutter build apk --debug
```

## شرح خوارزمية الباقات

`PackageOptimizer` يستخدم Dynamic Programming على إجمالي الرصيد الممكن حتى أول نطاق كافٍ فوق الهدف. لكل إجمالي رصيد يحتفظ بأفضل مرشح، ثم يقارن المرشحين النهائية بهذا الترتيب:

1. التكلفة الإجمالية الأقل.
2. الرصيد الزائد الأقل.
3. عدد الباقات الأقل.
4. أقصر صلاحية أفضل، ثم مجموع صلاحيات أفضل عند استمرار التعادل.

لا توجد شروط خاصة بـ 4080 أو 2400؛ تعديل الأسعار أو الباقات يغير النتيجة تلقائيًا.

## المخزون وFEFO

كل باقة مشتراة تنشئ `InventoryLot` مستقلًا يحتوي على:

- بيانات الباقة Snapshot.
- الرصيد الأصلي والمتبقي.
- التكلفة.
- وقت الشراء والانتهاء.
- الحالة.

أثناء الحفظ:

1. يستهلك الرصيد القديم الفعّال حسب أقرب انتهاء.
2. ينشئ رزم الباقات الجديدة.
3. يستهلك الجزء المطلوب من الرزم الجديدة.
4. يترك الفائض في المخزون.
5. يسجل كل حركة دخول وخروج.

عند حذف عملية يعاد بناء المخزون من العمليات المتبقية بالترتيب الزمني، بدل إجراء تعديل عكسي قد يفسد عمليات لاحقة.

## الربح
